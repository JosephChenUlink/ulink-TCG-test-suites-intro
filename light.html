ULINK • TCG Test Suites Introduction v1.0.0

ULINK TCG Test Suites

ULINK’s TCG test suites help drive makers and ecosystem partners validate TCG-compliant self-encrypting drives (SEDs) with repeatable, automation-driven testing. Choose from certification-oriented coverage, protocol verification, and workflow-based application note suites across Opal-family and Enterprise SSC devices.

🔎
What you get

Faster, repeatable TCG validation

DriveMaster automation Opal-family & Enterprise NVMe / SATA / SAS
  • Certification coverage to help validate Opal-family behavior and key feature sets.
  • Protocol verification to check message grammar, table content, and expected device behavior.
  • Application note workflows to validate practical “how it’s used” scenarios (ownership, locking, revert, and more).
  • Flexible execution via full-suite runs or targeted standalone scripts.

Suite overview

Below is a high-level snapshot of each suite and where it fits in a validation plan. (Details vary by release.)

Certification

SWG TCG Opal Family Certification Test Suite

Opal-family Certification cases

Certification-oriented coverage built around Opal-family test cases and common feature sets (e.g., PSID, Additional DataStore Tables, Block SID Authentication), plus interface interaction requirements.

Focus: Certification Devices: Opal-family SEDs Interfaces: ATA / SCSI / NVMe
Protocol

TCG Enterprise SSC Protocol Test Suite

Enterprise SSC Protocol verification

Protocol verification for Enterprise SSC devices. Organized into test groups for generic checks (e.g., Trusted Send/Receive, Discovery, Transactions), grammar checks (Get/Set/Next/GetACL), table-content checks, and effect checks.

Focus: Protocol Devices: Enterprise SSC SEDs Run levels: Main + standalone
Protocol

TCG/IEEE1667 Opal Family SSC Protocol Test Suite

Opal-family IEEE 1667

Protocol verification for Opal-family SSC devices with additional IEEE 1667 coverage, including feature checks for both TCG and IEEE1667 and OPALv1/OPALv2 script catalogues.

Focus: Protocol + IEEE1667 Devices: Opal-family SEDs Catalogues: OPALv1 / OPALv2
Protocol

TCG Opal Family SSC Multiple Namespaces Protocol Test Suite

NVMe namespaces Configurable locking

Protocol coverage for Opal devices implementing configurable locking for NVMe namespaces (and SCSI LUNs), including namespace locking examples and related Opal feature sets.

Focus: Protocol Key: Multi-namespace locking Platform: DriveMaster 9+
Application Note

TCG Opal Application Note Test Suite

Opal SSC Workflow-based

Workflow-based validation for Opal-family SEDs, aligned with TCG application note scenarios (ownership, Locking SP activation, locking ranges, MBR shadowing, revert, DataStore usage, and more).

Focus: AppNote workflows SSC: Opal-family Run levels: Main + standalone
Application Note

TCG Enterprise Application Note Test Suite

Enterprise SSC Workflow-based

Workflow-based validation for Enterprise SSC SEDs, aligned with TCG application note scenarios focused on practical integration and interoperability behaviors.

Focus: AppNote workflows SSC: Enterprise Run levels: Main + standalone
